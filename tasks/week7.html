Week 7 Tutorial: Class Design

Class Structure

A class can be thought of as a blueprint for a particular type of object. All objects of a certain type have characteristics (data, or state) and behavior (methods) in common, and so it is convenient to abstract these qualities into a class.

You've been using classes since you first started using python; addition in IDLE makes use of the int class, working with strings involves the str class, lists are based on the list class, and so on.

We can define our own classes in a similar way to defining functions. For example, a basic outline of a Student class might be as follows: 

class Student(object):
    def __init__(self, name):
        self.setName(name)

    def get_name(self):
        return self._name

    def set_name(self, name):
        self._name = name

Interacting with classes is also quite simple. Here is an example of how to create an instance of the Person class: 

>>> peter = Student('Peter Griffin')
>>> peter
<__main__.Student object at 0x103961150>
>>> peter.get_name()
'Peter Griffin'
