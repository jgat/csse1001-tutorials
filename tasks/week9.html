Week 9 Tutorial: Introduction to GUI Design

Background Information

If you are writng a program that is going to be used by other people, then you will most likely want to include a GUI (graphical user interface). Python ships with a GUI toolkit known as Tkinter; this is what IDLE is written in.

In this course, we will use Tkinter to introduce you to the basics of GUI design. An excellent resource on this toolkit can be found at http://effot.org/tkinterbook/.

An interface in Tkinter is built from a collection of widgets. Each widget is a Tkinter class that is designed for a specific purpose: the Button widget is a standard button, the Label widget a tool for displaying static text, the Frame widget a container for other widgets, and so on.

These widgets are arranged on-screen through what is known as a geometry manager. We will be using the pack geometry manager. It's probably easiest to think about pack as creating 'stacks' of widgets.

When you pack a widget, you tell Tkinter which side of the window you want to place it against. The first widget packed to a particular side is placed right against that edge of the window. Subsequent widgets are packed up against this one.

Displaying a Widget

There are two steps in creating a GUI widget in Tkinter. Firstly, the widget itself must be created, as an instance of the appropriate class. Any configuration information information relevant to the widget, such as the text it should display or the callback function it should run when interacted with, is given here as a keyword argument.
For example, this is how one might choose to create an instance of the Button widget:

btn = Button(self, text="I'm a button", command=self.print_hello)

Secondly, the widget itself must be packed. Keyword arguments to the pack method control how and where the widget is displayed on the screen. These arguments should never be given to the class directly, as some online examples won't do.
Packing the Button widget can be done like so:

btn.pack(side=TOP)

It is possible to position the widgets with more precision by making use of the anchor keyword arguments. Widgets can be anchored to compass directions (e.i. N, S, E and W). For example, the following code would pack btn in the top left (provided it was the first widget to be packed): 

btn.pack(side=TOP, anchor=N)

Frames

It's not very satisfactory to just stack widgets from the top, bottom or side of the screen; such an approach lacks flexibility (for example, to place two buttons side-by-side beneath a label).

We can overcome this limitation by creating frames to store widgets in, and then arranging these frames and their component widgets appropriately using pack.

Every widget in Tk has the concept of a master. You may have noticed that when we created the buttons and labels in the previous example, the first argument we provided to their contructor was self. This is the button or label's master.

Each widget is drawn inside of its master. When we pack a widget, it is packed inside and with respect to this master. What this means is that when we pack the widget with, say, side=TOP, it from the top of this master widget.

Interacting with Widgets

You will often want to interact with widgets after you have created them. For example, you often need to change the text that is displayed in a Label, or get information that a user has entered in an Entry (textbox).

It is possible to reconfigure tany property of a Tk widget by using the configure method. This method accepts the same keyword arguments as the constructor for a widget. For example, the following code will change the text of the label self._lbl.

self._lbl.configure(text="I've been changed!")

Controls that accept user input will define methods for accessing that input. For example, to get the contents of a textbox, you can use the Entry.get method.

Message Boxes in Tkinter

Occasionally a user will interact with a program in a way that requires a pop-up message box. This can be achieved in Tkinter by using the tkMessageBox module. For now, we will only concern ourselves with the module's showerror() function. The call required to obtain an error message box is of the form: 

import tkMessageBox
tkMessageBox.showerror(title, message [, options])
The title and message arguments dictate the text that will appear in the error box.
